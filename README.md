# dfad (Android Testing MCP Server)

このプロジェクトは、AIコーディングエージェント（Claude Code、Cursor、Windsurfなど）に**Android端末（実機・エミュレータ）を自律的に操作・検証させるためのMCP(Model Context Protocol) サーバー**です。

システム上の `adb` コマンドを内部でラップしており、AIエージェント自らが画面のスクリーンショットやUIツリー構造（View Hierarchy）を取得し、解析した座標をもとにタップやテキスト入力などの操作を行えるようにします。さらに、Logcatを絡めた内部状態の検証や、テストエビデンスとしての動画録画機能も備えています。

## 主な機能（提供されるツール一覧）

AIエージェントに以下のネイティブツール（能力）を提供します：

### 📱 画面操作・認識ツール
| ツール名 | 説明 |
| :--- | :--- |
| `get_screen_state` | 現在の画面のスクリーンショット(Base64画像)とUI構造(XMLダンプ)を一緒に取得します。画面状態の認識に使用します。 |
| `get_top_activity` | 画面の最前面で実行されているパッケージ名とActivity情報を取得します。想定外の画面にいないかの即座な確認に便利です。 |
| `tap` | 画面の指定された座標 (X, Y) をタップします。 |
| `swipe` | 画面上の指定された座標間を時間(ms)をかけてスワイプ/スクロールします。長押しにも利用可能です。 |
| `input_text` | フォーカスされている入力欄にテキストを入力します。 |
| `key_event` | ホームボタン(3)、戻るボタン(4)、エンターキー(66)などのシステムキーを発行します。 |
| `launch_app` | 指定したパッケージ名のAndroidアプリを起動します。 |

### 🔍 検証・エビデンスツール
| ツール名 | 説明 |
| :--- | :--- |
| `clear_logcat` | Logcatバッファをクリア(`adb logcat -c`)します。テスト開始前に呼ぶことで古いログの混入を防ぎます。 |
| `get_logcat` | 直近のLogcatを取得します。特定文字列でのフィルタリング(`filterText`)も可能で、API通信や例外エラー(`FATAL EXCEPTION`)の検証に使います。 |
| `start_recording` | 画面録画をバックグラウンドで開始します。(動画はトークン節約のためAIには渡りません) |
| `stop_recording` | 録画を停止し、PC側の `records/` フォルダ内にmp4ファイルとして自動保存します。人間が見るためのテストエビデンス用途です。 |

---

## システム要件

1. **Node.js**: v18以上を推奨
2. **Android SDK Platform-Tools (`adb`)**: ターミナル上でグローバルに `adb` コマンドが実行可能であり、対象のAndroid端末やエミュレータが認識されている（`adb devices` で表示される）こと。

---

## セットアップ方法

### 1. npx を使ったグローバル利用 (おすすめ)
どのプロジェクトディレクトリにいてもこのMCPサーバーを実行したい場合は、ご自身のAIクライアントの設定(mcp.json等)に以下を追記してください。
```json
{
  "mcpServers": {
    "dfad": {
      "command": "npx",
      "args": ["-y", "github:slme9364/dfad"]
    }
  }
}
```

### 2. ローカルでの対話型インストール
リポジトリをクローンして手元で動かす場合や、ワンタッチでClaude Code等に登録したい場合は以下のコマンドをご利用ください。

```bash
# プロジェクトの依存関係をインストール
npm install

# 対話型インストールウィザードを起動
npm run install:mcp
```
> ウィザード内で**「1. Claude Code (CLI)」**を選ぶと、`-s user` オプションを使ってあなたのPC全体（グローバルスコープ）で有効になるよう自動登録されます。

## 利用方法（プロンプトの例）

セットアップ完了後、AIエージェントと会話を始めて以下のように指示を出すだけで、録画・検証を含めた自律的なAndroidテストがスタートします。

> **AIへのプロンプト例（フル機能テスト）:**  
> 「ターミナルで `./gradlew installDebug` を実行してアプリをインストールした後、`launch_app` でアプリを起動して。
> 次に `start_recording` で録画を開始し、`clear_logcat` を叩いてから、現在の画面を `get_screen_state` で確認してログインを試みて。
> ログイン完了後、`get_logcat` で通信エラーや例外が出ていないか確認し、すべて終わったら `stop_recording` で動画を保存してください。」
